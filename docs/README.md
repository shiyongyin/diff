# 文档索引

本目录包含 Tenant Diff 的运行时 SSOT 文档与补充性评估/规划文档。

## 运行时 SSOT 文档

PRD、设计、运维为运行时 SSOT（单一权威源），描述当前接口合约、实现行为和操作规程。

| 文档 | 定位 | 目标读者 |
|------|------|---------|
| [产品需求文档 (PRD)](prd.md) | 产品概述、快速体验、功能规格、API 合约、业务规则、版本策略 | 产品经理、前端开发、实施顾问 |
| [开发设计文档](design-doc.md) | 架构、领域模型、核心流程、SPI 扩展点、插件开发指南、数据库设计 | 后端开发、架构师 |
| [运维手册](ops-guide.md) | 部署、配置、监控、故障排查、发布管理、安全运维、CI/CD | 运维工程师、SRE |

## 补充文档（阶段性分析材料）

以下文档为阶段性评估和规划，不作为接口/行为的唯一事实来源。部分内容可能滞后于实际实现，以运行时 SSOT 文档为准。

| 文档 | 定位 | 目标读者 |
|------|------|---------|
| [项目质量评估](project-assessment.md) | 全方位质量评估、差距分析、优势与风险（基于特定时间点的快照） | 项目负责人、技术决策者 |
| [演进路线](evolution-roadmap.md) | 功能/安全/工程/生产各阶段演进计划、版本里程碑（部分规划项可能已完成） | 项目负责人、研发团队 |

## 推荐阅读顺序

1. **首次接触**：先看 `prd.md` 的"快速体验"章节
2. **开发扩展**：看 `design-doc.md` 的"插件开发指南"章节
3. **生产部署**：看 `ops-guide.md` 的"部署指南"和"故障排查"章节
4. **项目全局**：看 `project-assessment.md` 了解当前状态，然后看 `evolution-roadmap.md` 了解下一步
