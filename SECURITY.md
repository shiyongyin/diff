# Security Policy

## 支持版本

当前仓库仍处于 `0.x` 阶段，原则上只保证最新开发分支和最近一次发布版本的安全修复。

| Version | Supported |
|---------|-----------|
| `0.0.x` | Yes |
| `<0.0.1` | No |

## 报告方式

请 **不要** 通过公开 Issue 披露安全漏洞。

推荐流程：

1. 如果仓库托管平台支持私有安全报告，请优先使用私有报告渠道。
2. 如果暂时没有私有报告能力，请先通过维护者私有渠道联系，不要公开披露利用细节。
3. 报告中请尽量包含：影响范围、复现步骤、可能后果、修复建议、是否已有临时缓解方案。

## 响应目标

在维护资源允许的前提下，项目希望达到以下响应节奏：

- `3` 个工作日内确认收到报告
- `7` 个工作日内给出初步评估
- 修复完成后尽快发布安全说明或修复版本

## 披露原则

在补丁可用前，请避免公开发布可直接利用的细节。

项目维护者会在修复后尽量提供：

- 受影响版本范围
- 风险描述
- 修复方式或缓解方案
- 是否存在兼容性影响
